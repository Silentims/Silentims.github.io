与前缀和的思路差不多，学习到了差分数组：

差分数组的应用场景是，需要对某个区间 [i...j] 频繁地加或减某一值，避免每次都遍历这个区间。比如拼车的题中，既有上车，也有下车，所以可以遍历每个站点，上车的加人，下车的减人。而航班预定的题思路则是，遍历每一个航班，涉及出航定订单加，后边不涉及的减，本质上也是利用分差数组，即求区间 [i,j]，则从 i 开始往后的求和，减去 j+1 往后的求和，就是 i 到 j 之间的和。

# Day05 二分搜索原理

今天的课程是「day5：二分搜索原理」

必做作业有 1 题：

704. 二分查找（简单）open in new window

选做题 1 道：

34. 在排序数组中查找元素的第一个和最后一个位置（中等）open in new window

  • 整体感受

技巧汇总：

使用左闭右闭的方式时，有几个需要注意的点：

  1. rigth = nums.length - 1
  2. while(left <= right) 因为右闭，所以不能漏掉最后一个元素，此处是有“=”的
  3. int mid = left + (right - left) / 2; 等同于 (left + right) / 2；并且可以避免数值越界
  4. nums[mid] < target 则移动 left，相反则移动 right

附 :

二分法写法框架
//全部采用左闭右闭的写法
//普通二分查找
int binary_search(int[] nums, int target) {
    int left = 0, right = nums.length - 1; 
    while(left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1; 
        } else if(nums[mid] == target) {
            // 直接返回
            return mid;
        }
    }
    // 直接返回
    return -1;
}

//寻找左侧边界的二分查找，在最后需要检查left的越界情况
int left_bound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] == target) {
            // 别返回，锁定左侧边界
            right = mid - 1;
        }
    }
    // 最后要检查left越界的情况
    if (left >= nums.length || nums[left] != target)
        return -1;
    return left;
}

//寻找右侧边界的二分查找，在最后需要检查right的越界情况
int right_bound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] == target) {
            // 别返回，锁定右侧边界
            left = mid + 1;
        }
    }
    // 最后要检查 right 越界的情况
    if (right < 0 || nums[right] != target)
        return -1;
    return right;
}

# Day07 滑动窗口技巧

今天的课程是「day7：滑动窗口技巧」

必做作业有 1 题：

3.无重复字符的最长子串（中等）open in new window

选做题 3 道：

567.字符串的排列（中等）open in new window

438.找到字符串中所有字母异位词（中等）open in new window

76.最小覆盖子串（困难）open in new window

  • 思路
  1. 定义[left,right)的滑动窗口左闭右开区间
  2. 向右滑动right，寻找一个「可行解」
  3. 向右滑动left，优化「可行解」
  • 代码框架
第 567 题案例
class Solution {
    public boolean checkInclusion(String s1, String s2) {
        
        // 整体思路就是：左闭右开区间，双指针，先扩大右指针，直到窗口符合要求，然后缩小左指针，直到窗口不满足要求，此时窗口中就是所求的字串

        /*
         * 具体思路：
         * 1. 先将需要匹配的字符串放入need的HashMap中
         * 2. 移动右指针，将符合的字符放入窗口中，直到满足全部字串字符
         * 3. 移动左指针，直到窗口中字符不满足停下
         * 4. 此时窗口中右指针 - 左指针 + 1，则为对应的字串
         */

        /* 滑动窗口框架 */

        //1.定义窗口，need为符合条件的窗口，window为滑动窗口
        Map<Character, Integer> need = new HashMap<>();
        Map<Character, Integer> window = new HashMap<>();

        //2.将need要检查的字符串放入map中
        for (char c : s1.toCharArray()) {
            need.put(c, need.getOrDefault(c, 0) + 1);
        }

        //3.初始化左右指针和valid
        int left = 0, right = 0;
        int valid = 0;

        /*
         * 滑动窗口前，需要考虑的4个问题
         * 1、当移动 right 扩大窗口，即加入字符时，应该更新哪些数据？window++，valid++
         * 2、什么条件下，窗口应该暂停扩大，开始移动 left 缩小窗口？当Valid == need中元素的个数
         * 3、当移动 left 缩小窗口，即移出字符时，应该更新哪些数据？window--，valid--
         * 4、我们要的结果应该在扩大窗口时还是缩小窗口时进行更新？缩小窗口
         */
        //4.扩大窗口
        while (right < s2.length()) {
            // c 是将移入窗口的字符
            char c = s2.charAt(right++);//right++表示先赋值，后运算
            
            // 进行窗口内数据的一系列更新
            if (need.containsKey(c)) {
                window.put(c, window.getOrDefault(c, 0) + 1);
                if (window.get(c).equals(need.get(c)))//这一步是校验need和window中是否都有字符c，有说明匹配
                    valid++;
            }

            // 5.缩小窗口，判断左侧窗口是否要收缩
            while (right - left == s1.length()) {
                // 在这里更新最小覆盖子串
                if (valid == need.size()) {
                    return true;
                }
                // d 是将移出窗口的字符
                char d = s2.charAt(left++);

                // 进行窗口内数据的一系列更新
                if (need.containsKey(d)) {
                    if (window.get(d).equals(need.get(d)))
                        valid--;
                    window.put(d, window.get(d) - 1);
                }
            }
        }
        return false;
    }
}

单调栈框架
import java.util.Arrays;
import java.util.LinkedList;

// 单调栈模板
// 每个元素最多入栈、出栈一次，因此时间复杂度为O(N)，空间复杂度O(N)
public class Module {
    // 获取每个元素右侧的较大值
    public static int[] nextGreaterElement(int[] nums) {
        // 记录较大值的值
        int[] res = new int[nums.length];
        // 维持递减单调栈 
        LinkedList<Integer> stack = new LinkedList<>();
        // 从右往左，考虑右边的较大值
        for (int i = nums.length - 1; i > -1; i--) {
            // 弹出较小值
            while (stack.size() != 0 && stack.peek() <= nums[i]) {
                stack.pop();
            }
            // 获取较大值
            res[i] = stack.size() == 0 ? -1 : stack.peek();
            // 入栈, 以stack[0]为栈顶,
            // 实际上调用了addFirst()
            // 使用add()则变成了队列，与其他方法操作的逻辑不同
            stack.push(nums[i]);
        }
        return res;
    }

    // 获取每个元素右侧较大值的索引
    public static int[] nextGreaterElementIdx(int[] nums) {
        // 记录较大值的索引
        int[] res = new int[nums.length];
        // 维持递减单调栈
        LinkedList<Integer> stack = new LinkedList<>();
        // 从右往左，考虑右边的较大值
        for (int i = nums.length - 1; i > -1; i--) {
            // 弹出较小值
            while (stack.size() != 0 && nums[stack.peek()] <= nums[i]) {
                stack.pop();
            }
            // 获取较大值的索引
            res[i] = stack.size() == 0 ? -1 : stack.peek();
            // 入栈, 以stack[0]为栈顶,
            // 实际上调用了addFirst()
            // 使用add()则变成了队列，与其他方法操作的逻辑不同
            stack.push(i);
        }
        return res;
    }

    // 获取每个元素循环右侧较大值
    public static int[] nextCyclicGreaterElement(int[] nums) {
        // 记录较大值
        int[] res = new int[nums.length];
        // 维持递减单调栈
        LinkedList<Integer> stack = new LinkedList<>();
        // 从右往左，考虑右边的较大值
        // 在右侧虚拟增加一个原数组
        for (int i = 2 * nums.length - 1; i > -1; i--) {
            // 弹出较小值
            while (stack.size() != 0 && stack.peek() <= nums[i % nums.length]) {
                stack.pop();
            }
            // 获取较大值
            res[i % nums.length] = stack.size() == 0 ? -1 : stack.peek();
            // 入栈, 以stack[0]为栈顶,
            // 实际上调用了addFirst()
            // 使用add()则变成了队列，与其他方法操作的逻辑不同
            stack.push(nums[i % nums.length]);
        }
        return res;
    }


    // 测试用例
    public static void main(String[] args) {
        System.out.println(Arrays.toString(nextGreaterElement(new int[] {2, 1, 2, 4, 3}))); // [4, 2, 4, -1, -1]
        System.out.println(Arrays.toString(nextGreaterElementIdx(new int[] {2, 1, 2, 4, 3}))); // [3, 2, 3, -1, -1]
        System.out.println(Arrays.toString(nextCyclicGreaterElement(new int[] {2, 1, 2, 4, 3}))); // [4, 2, 4, -1, 4]
    }
}

  • 技巧

快速排序就是个二叉树的前序遍历，归并排序就是个二叉树的后序遍历

写递归算法的关键是要明确函数的「定义」是什么，然后相信这个定义，利用这个定义推导最终结果，绝不要跳入递归的细节。

写树相关的算法，简单说就是，先搞清楚当前 **root** 节点「该做什么」以及「什么时候做」，然后根据函数定义递归调用子节点，递归调用会让孩子节点做相同的事情。
