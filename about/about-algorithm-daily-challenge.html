# LeetCode 21 天挑战日志

写一个流水式日志，来记录自己在参加刷张组织的 21 天算法挑战过程中，刷题遇到的问题和一些想法：

# 为什么参加算法挑战 ？

  1. 算法一直是自己久攻不下和很难坚持下去的弱项。
  2. 许多大厂目前基本上都要考算法，中小厂也会可能有算法题。
  3. 学习算法，养成算法的思维，我觉得可以提升计算机思维，从计算机的角度思考问题，在日后的实际工作中也能更好的选用不同的数据结构来实现逻辑，提升对数据结构的理解。
  4. 正好有这么个免费参加的机会，当然要把握住机会。

# Day01 数组 (2021-08-19)

详情

第一周都是数组。

今天是第一天，课程内容是「原地修改数组」，课程链接：《day1：原地修改数组》open in new window

必做作业有 2 题：

26.删除排序数组中的重复项（简单）open in new window

刚开始没什么思路，稍微看了一眼题解思路后，发现用的是“双指针”，然后自己实操了一下，发现编码还是不通过，最后经过好几次调整代码才过，主要的问题就是判断条件判断不准；

27.移除元素（简单）open in new window

选做题有 2 题：

283.移动零（简单）open in new window

83.删除排序链表中的重复元素（简单）open in new window

  • 整体感受：

首先 4 题中有 2 题在 4 个月前刷过，分别是 27 和 283，即移除元素和移动零，虽然刷过，但是看完题目，依旧思路不是很清晰，所以简单看了下课件后，知道了双指针，所以后面 解题思路就清晰一些，但是实际编码的质量不高，还会存在语法的问题，其次判断条件的临界值没弄清楚，导致频繁出现数组越界的问题。

  • 主要收获：

数组双指针（快慢指针，左右指针）的解题思路；
