# 关于 - Algolia DocSearch 的使用（未开放）

Algolia DocSearch 是一个适用于文档网站的搜索工具，它可以根据网站的标题和内容，创建搜索索引来支持对网站中内容的全站搜索，Vuepress 官方已经集成了该插件，由于网站初始内容不较少，无法通过官方帮助爬取来获得 API Key，所以需要自行注册和爬取网站来创建索引，以下是个人按照官方文档说明成功爬取网站，生成索引和使用插件的操作流程。

提示

以下内容均在云服务器和 MacOS 上实践过,本篇主要以云服务器操作演示

# 注册与登录

打开Algolia DocSearch官网open in new window注册，亦可通过 Github 等第三方账号授权登录

# 生成 API Key

登录成功后，进入「Dashboard」控制后台，在「API Keys」中，可以查看自己的Application ID和许多带有不同权限的API Key，由于我们是自行创建搜索索引，所以此处按照官方说明文档open in new window，需要一个包含addObject、editSettings、deleteIndex权限的密钥，所以先创建具有这三个权限的 API Key

# 参数配置

主要包括两部分参数：

  1. .env配置文件

在电脑上根目录新增.env文件，配置如下信息并保存

APPLICATION_ID=你的Application ID
API_KEY=上述中在管理后台生成的 API Key
1
2
  1. config.json爬取脚本文件

按下面配置文件进行适当的修改，其中index_name为索引名，免费版默认都使用 example

{
  "index_name": "example",
  "start_urls": [
    "https://www.techxiang.com/"
  ],
  "stop_urls": [],
  "selectors": {
    "lvl0": {
      "selector": ".sidebar-heading.active",
      "global": true,
      "default_value": "Documentation"
    },
    "lvl1": ".theme-default-content h1",
    "lvl2": ".theme-default-content h2",
    "lvl3": ".theme-default-content h3",
    "lvl4": ".theme-default-content h4",
    "lvl5": ".theme-default-content h5",
    "text": ".theme-default-content p, .theme-default-content li",
    "lang": {
      "selector": "/html/@lang",
      "type": "xpath",
      "global": true
    }
  },
  "custom_settings": {
    "attributesForFaceting": ["lang"]
  }
}

# 爬取索引

本文主要通过 docke 创建官方的镜像进行爬取

确保爬取前外网可访问的网站中已经是最新的文章

# 安装EPEL源：
yum install epel-release

# 安装完EPEL源后，可以查看下jq包是否存在：
yum list jq

# 安装jq：
yum install jq
