# IDEA 阅读和调试 jdk8 源码

写在前面

目的：为了更好的阅读和学习 Java 底层源码

说明

  • MacOS 是通过 HomeBrew 安装的 openJDK8
  • Windows 系统可参考文中最后的链接教程

# 准备

找到电脑中安装 jdk8 的位置，本篇需要的源码就是安装目录中的 src.zip

# 创建项目

打开 idea，创建一个简单的 Java 项目

创建一个命令行的简单项目（只有一个 main 方法）

自定义项目名，项目位置，包名

打开创建的项目的实际位置，并将 src.zip 解压到项目中的 src 中即可，解压完后项目目录如下（图中是已经移除报错不要的部分，后面的问题会说到具体删除了哪些文件）

# 配置与添加依赖

进入设置，调整编译的堆大小，默认为 700，此处调整为 1700（避免启动编译默认的堆内存不足导致启动失败）

打开 File--Project Structure--Libraries，添加依赖（即引用原安装的 jdk 中的 lib 文件夹下的 jar 包），

  2. 把 src/sun 文件夹整体删除；

经过查阅很多网上的问题解决办法，都没有查到我遇到的这个问题，倒是其他问题什么样的都有，所以可能每个人遇到的报错都不一样；

当然，总的来说，只是为了学习 Java 开发工具包中的源码，有一些底层、可视化或者系统支持的包实际上是不需要的，可以根据实际酌情删除后，反复启动项目来排查问题。

# 源码注释

项目启动成功后，在 main 方法中写一个简单的代码

public class Main {

    public static void main(String[] args) {
				// write your code here
        HashMap<String, String> map = new HashMap<>();
        map.put("AAA", "aaa");
    }
}

此时点击 put 方法进入会发现，是 .class 文件，不可编辑，因为项目引入的 SDK 是安装的 jdk 目录，为了能点击进入到的是本项目的 .java 文件位置，同时如果想添加源码注释，就需要修改 SDKs 目录：

打开 File--Project Structure--Libraries，新增一个 SDKs 配置

注意：图中 1.8src 就是新增的配置，源码路径为当前项目的路径，新增的原因是避免直接修改默认的配置可能会导致别的项目出现引用错误

修改本项目引用的 sdk 为 1.8_src，保存后，再次点击代码跳转，就是本项目的实际 java 文件位置了

不管是点击方法看源码，或者debug进入源码，都是当前项目的.java文件

# 正确的食用方法

  1. 在需要调试的源码入口打断点；
  2. 若需要对源码进行添加注释，一定要在当前行备注，避免换行导致编译执行报错与实际代码位置不符的问题，不方便排查
